## Overview

Servineo provides a comprehensive payment system for requesters to pay for completed services. The platform supports QR code payments, wallet management, and detailed transaction history.

## Payment Methods

### QR Code Payment

The primary payment method using QR codes for secure transactions.

#### Payment Flow

<Steps>
  <Step title="Service Completion">
    Fixer marks the job as completed
  </Step>
  <Step title="Payment Intent">
    System creates a payment intent with amount and details
  </Step>
  <Step title="QR Generation">
    Backend generates a unique QR code for the transaction
  </Step>
  <Step title="Scan & Pay">
    Scan QR with your banking app to complete payment
  </Step>
  <Step title="Confirmation">
    Payment is confirmed and fixer receives funds
  </Step>
</Steps>

#### Payment Intent

```typescript
interface PaymentIntent {
  _id: string;
  bookingId: string;
  providerId: string;
  amountExpected: number;
  currency: string;
  paymentReference: string;
  status: PaymentStatus;
  deadlineAt?: string;
  createdAt?: string;
}

type PaymentStatus = 
  | 'pending'       // Awaiting payment
  | 'under_review'  // Payment received, verifying
  | 'confirmed'     // Payment confirmed
  | 'rejected'      // Payment failed or rejected
  | 'expired';      // Payment deadline passed
```

#### QR Payment Page

The QR payment interface (`/payment/pay-with-qr`) displays:

**Payment Information:**
- **Destinatario**: Fixer's account details
- **Nro de Transacción**: Unique payment reference

- **Comisión**: Platform fee (3% of total)

**Payment Status:**

- Deadline information
- Transaction history

The QR image is loaded from:

```typescript
interface PaymentMethod {

// Handles Google Drive URLs
const toDriveThumb = (url: string) => {
  const m = url.match(/id=([^&]+)/);
  return m ? `https://drive.google.com/thumbnail?id=${m[1]}&sz=w512` : url;
};

// Fallback QR if image fails to load
const fallbackQR = `https://api.qrserver.com/v1/create-qr-code/?size=256x256&data=${encodeURIComponent('Servineo QR')}`;
```

<Warning>
If the QR code doesn't load properly, the system will attempt to convert Google Drive URLs to thumbnail format or display a fallback QR.
</Warning>

#### Date Validation

The filter system validates dates thoroughly:

```typescript
const validateDateString = (dateStr: string, fieldName: string): string | null => {
  if (dateStr.length !== 10) {
    return `La fecha "${fieldName}" esta incompleta. Use formato MM/DD/YYYY`;
  }
  
  const parts = dateStr.split('/');
  const month = parseInt(parts[0], 10);
  const day = parseInt(parts[1], 10);
  const year = parseInt(parts[2], 10);
  
  if (!isValidDate(month, day, year)) {
    if (month === 2 && day > 29) {
      return `Fecha invalida: Febrero solo tiene hasta 29 dias`;
    }
    return `La fecha "${fieldName}" no existe en el calendario`;
  }
  
  return null;
};
```

<Info>
Date filters use MM/DD/YYYY format with automatic slash insertion. You can also use the calendar picker for easier selection.
</Info>

### Payment Breakdown

Typical payment structure:

```typescript
const calculatePaymentBreakdown = (total: number) => {
  const commission = Math.round(total * 0.03);  // 3% platform fee
  const subtotal = total - commission;          // Amount to fixer
  
  return {
    subtotal,
    commission,
    total
  };
};

// Example:
// Total: Bs. 215.00
// Subtotal: Bs. 208.55 (to fixer)
// Commission: Bs. 6.45 (platform fee)
```

<Warning>
All payments include a 3% platform commission. This fee covers transaction processing, platform maintenance, and support services.
</Warning>

## Troubleshooting

### Common Issues

#### QR Code Won't Load

**Symptoms:**
- QR code shows error or doesn't display
- Image placeholder appears

**Solutions:**
1. Refresh the page
2. Check internet connection
3. Clear browser cache
4. Try different browser
5. Contact support if issue persists

```typescript
// QR loading with fallback
<img
  src={imgSrc}
  alt="QR de pago"
  onError={() => {
    if (imgSrc.includes('drive.google.com') && !imgSrc.includes('/thumbnail')) {
      setImgSrc(toDriveThumb(imgSrc));
    } else {
      setImgSrc(fallbackQR);
    }
  }}
/>
```

#### Payment Not Confirming

**Symptoms:**
- Paid but status still shows pending
- Payment completed in bank but not in Servineo

**Solutions:**
1. Wait 5-10 minutes for confirmation
2. Check payment reference matches
3. Verify amount is correct
4. Contact support with reference number

#### Transaction Not in History

**Symptoms:**
- Recent payment doesn't appear in history
- Missing transactions in list

**Solutions:**
1. Refresh the page
2. Clear date filters
3. Check transaction type filters
4. Wait for sync (up to 5 minutes)

<Info>
If you experience payment issues, note your payment reference number and contact support immediately. Keep your banking app receipt as proof of payment.
</Info>

## Best Practices

### Before Payment

1. **Verify Amount**: Check total matches agreed price
2. **Review Details**: Confirm job description and fixer
3. **Check Balance**: Ensure sufficient funds in bank account
4. **Save Reference**: Note payment reference before scanning

### During Payment

1. **Scan Carefully**: Ensure QR code is fully visible
2. **Verify Details**: Check amount in banking app matches
3. **Complete Promptly**: Don't let QR expire
4. **Save Receipt**: Keep bank transaction confirmation

### After Payment

1. **Wait for Confirmation**: Allow time for processing
2. **Check Status**: Verify payment status updates
3. **Download Receipt**: Save invoice for records
4. **Leave Review**: Rate the fixer's service
