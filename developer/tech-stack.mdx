## Core Framework

### Next.js 15.5.4

Servineo uses the latest Next.js 15 with the **App Router** architecture (not Pages Router).

```json package.json
{
  "dependencies": {
    "next": "15.5.4",
    "react": "19.1.0",
    "react-dom": "19.1.0"
  }
}
```

**Key Features Utilized:**

- **Turbopack**: Fast bundler for development and production builds
- **App Router**: File-system based routing with `src/app/` directory
- **Server Components**: React Server Components by default
- **API Rewrites**: Proxy configuration for backend API calls
- **Image Optimization**: Next.js Image component with remote patterns

<Info>
  Turbopack is enabled in both dev and build scripts (`--turbopack` flag), providing significantly faster compilation than webpack.
</Info>

### React 19.1.0

Latest React version with improved performance and new features:

- Server Components support
- Enhanced concurrent rendering
- Automatic batching improvements
- New hooks and APIs

<Warning>
  React 19 is a major version with breaking changes. Ensure all dependencies are compatible with React 19 before upgrading further.
</Warning>

## State Management

### Redux Toolkit 2.10.1

Modern Redux with simplified APIs and best practices built-in.

```typescript src/app/redux/store.ts
import { configureStore } from '@reduxjs/toolkit';
import { baseApi } from './services/baseApi';
import userReducer from './slice/userSlice';
import fixerReducer from './slice/fixerSlice';
import filterReducer from './slice/filterSlice';

export const store = configureStore({
  reducer: {
    [baseApi.reducerPath]: baseApi.reducer,
    user: userReducer,
    fixer: fixerReducer,
    filters: filterReducer,
    jobOffers: jobOffersReducer,
    jobOfert: jobOfertReducer,
    fixersByJob: fixersByJobReducer,
  },
  middleware: (getDefaultMiddleware) =>
    getDefaultMiddleware().concat(baseApi.middleware).concat(logger),
  devTools: process.env.NODE_ENV !== 'production',
});
```

**Active Slices:**

- `user` - User authentication and profile (37 lines)
- `fixer` - Service provider data (37 lines)
- `filters` - Search and filter state (125 lines)
- `jobOffers` - Job listings (40 lines)
- `jobOfert` - Individual job details (295 lines)
- `fixersByJob` - Fixer associations (43 lines)

### RTK Query

Data fetching and caching layer built on Redux Toolkit:

```typescript src/app/redux/services/baseApi.ts
import { createApi, fetchBaseQuery } from '@reduxjs/toolkit/query/react';

export const baseApi = createApi({
  reducerPath: 'api',
  baseQuery: fetchBaseQuery({
    baseUrl: process.env.NEXT_PUBLIC_API_URL + '/api',
    credentials: 'include',
  }),
  tagTypes: [
    'User', 'Job', 'Statistics', 'JobOffer', 
    'Requester', 'SearchHistory', 'Experience', 
    'Portfolio', 'Certification'
  ],
  endpoints: () => ({}),
});
```

**API Services** (extended from baseApi):

- `loginApi` - Authentication
- `userApi` - User management
- `jobApi` / `jobOfferApi` / `jobOffersApi` - Job operations
- `fixerApi` - Service provider endpoints
- `searchApi` / `searchHistoryApi` - Search functionality
- `dashboardApi` - Analytics and statistics
- `experiencesApi` / `portafolioApi` / `certifications` - Profile data

### Redux Logger

Development-time debugging for Redux actions:

```typescript
import logger from 'redux-logger';

middleware: (getDefaultMiddleware) =>
  getDefaultMiddleware()
    .concat(baseApi.middleware)
    .concat(logger)
```
