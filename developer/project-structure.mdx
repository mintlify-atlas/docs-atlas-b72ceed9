### App Router Key Files

<CodeGroup>

```tsx layout.tsx
// src/app/[locale]/layout.tsx
// Root layout with internationalization
export default async function RootLayout({ 
  children, 
  params 
}: Props) {
  const { locale } = await params;
  const messages = await loader();
  
  return (
    <html lang={locale}>
      <body>
        <NextIntlClientProvider locale={locale} messages={messages}>
          <Providers>
            <TourLogic />
            <ConditionalTopMenu />
            {children}
            <FooterSection />
          </Providers>
        </NextIntlClientProvider>
      </body>
    </html>
  );
}
```

```tsx providers.tsx
// src/app/providers.tsx
// Client-side provider wrapper
export function Providers({ children }: { children: React.ReactNode }) {
  return (
    <Provider store={store}>
      <TourProviderWrapper>
        {children}
      </TourProviderWrapper>
    </Provider>
  );
}
```

</CodeGroup>

<Note>
  All routes under `[locale]/` are automatically internationalized. The locale parameter is extracted from the URL (e.g., `/en/login`, `/es/login`).
</Note>

### Redux Store Structure

The store is organized into domain-specific slices:

```typescript src/app/redux/store.ts
export const store = configureStore({
  reducer: {
    [baseApi.reducerPath]: baseApi.reducer,  // RTK Query cache
    user: userReducer,                       // Auth & user data
    fixer: fixerReducer,                     // Fixer profiles
    filters: filterReducer,                  // Search filters
    jobOffers: jobOffersReducer,             // Job listings
    jobOfert: jobOfertReducer,               // Single job
    fixersByJob: fixersByJobReducer,         // Job-fixer relations
  },
  middleware: (getDefaultMiddleware) =>
    getDefaultMiddleware()
      .concat(baseApi.middleware)  // RTK Query middleware
      .concat(logger),              // Redux logger (dev only)
});
```

<Tip>
  Always use typed hooks from `src/app/redux/hooks.ts`:
  
  ```typescript
  import { useAppDispatch, useAppSelector } from '@/app/redux/hooks';
  
  const dispatch = useAppDispatch();
  const user = useAppSelector(state => state.user);
  ```
</Tip>

## Key Architectural Patterns

### 1. Feature-Based Organization

Redux features are organized by domain (jobOffers, searchHistory, etc.) with co-located hooks, selectors, and utilities.

### 2. Component Hierarchy

- **Atoms**: Smallest reusable components (`src/Components/atoms/`)
- **Shared**: Cross-feature components (`src/Components/Shared/`)
- **Feature**: Feature-specific components (calendar, fixer, etc.)
- **Page**: App Router page components (`src/app/[locale]/*/page.tsx`)

### 3. Layered Architecture

```
Presentation Layer     → src/Components/, src/app/[locale]/
Business Logic Layer   → src/app/redux/features/, src/hooks/
Data Access Layer      → src/app/redux/services/, src/services/
Utility Layer          → src/utils/, src/lib/
```

### 4. Path Aliases

```typescript tsconfig.json
{
  "paths": {
    "@/*": ["./src/*"]
  }
}
```

Always use `@/` for absolute imports:

```typescript
import { useAppSelector } from '@/app/redux/hooks';
import TopMenu from '@/Components/Navigation/TopMenu';
import { IUser } from '@/types/user';
```

## File Naming Conventions

- **Components**: `MyComponent.tsx` (PascalCase)
- **Hooks**: `useMyHook.ts` (camelCase)
- **Utils**: `myUtility.ts` (camelCase)
- **Types**: `my-type.ts` (kebab-case) or `MyType.ts` (PascalCase)
- **Pages**: `page.tsx` (Next.js convention)
- **Layouts**: `layout.tsx` (Next.js convention)

<Warning>
  Inconsistent casing exists in the codebase (e.g., `User.ts` vs `user.ts`). New files should follow lowercase or kebab-case for consistency and cross-platform compatibility.
</Warning>
