## Overview

The User Management system in Servineo allows administrators to monitor and manage all platform users, including requesters (service seekers) and fixers (service providers).

## User Statistics

The admin dashboard displays real-time user statistics fetched from the backend API:

### Fetching User Stats

```typescript /user-admin/lib/api.ts
getUserStats: async (token: string) => {
  console.log('📊 Obtaining real user statistics...');
  
  try {
    const response = await fetch(`${API_BASE}/api/admin/user-stats`, {
      headers: {
        Authorization: `Bearer ${token}`,
        'Content-Type': 'application/json',
      },
    });
    
    if (!response.ok) {
      throw new Error(`HTTP ${response.status}`);
    }
    
    const data = await response.json();
    return data;
  } catch (error) {
    // Fallback to cached data
    return {
      success: true,
      data: {
        totalUsers: 152,
        usersByRole: {
          requester: 110,
          fixer: 41,
          visitor: 0,
          admin: 1
        },
        timestamp: new Date().toISOString(),
        note: 'Backup data (connection failed)'
      }
    };
  }
}
```

<Info>
The API includes fallback data to ensure the dashboard remains functional even during connection issues.
</Info>

### Loading User Statistics

<Steps>
  <Step title="Authentication Check">
    Verify admin token validity before loading data
  </Step>
  <Step title="Fetch Real-Time Data">
    Request current user statistics from the backend API
  </Step>
  <Step title="Update Dashboard">
    Display metrics and update timestamp
  </Step>
  <Step title="Handle Errors">
    Fall back to cached data if API request fails
  </Step>
</Steps>

```typescript /user-admin/components/adminDashboard.tsx
const checkAuthAndLoadData = async () => {
  const token = localStorage.getItem('adminToken');
  const adminData = localStorage.getItem('adminUser');

  if (!token || !adminData) {
    router.push('/es/user-admin');
    return;
  }

  try {
    // Verify token
    const authCheck = await adminAPI.verifyToken(token);
    if (!authCheck.valid) {
      localStorage.removeItem('adminToken');
      localStorage.removeItem('adminUser');
      router.push('/es/user-admin');
      return;
    }

    // Load REAL user statistics
    console.log('🔄 Loading REAL statistics...');
    const statsResponse = await adminAPI.getUserStats(token);
    
    if (statsResponse.success && statsResponse.data) {
      setUserStats(statsResponse.data);
      setLastUpdated(new Date(statsResponse.data.timestamp).toLocaleTimeString());
      console.log('✅ REAL data loaded:', statsResponse.data);
    }
  } catch (error) {
    console.error('❌ Error loading data:', error);
    // Fallback data
  }
};
```

## User Roles

Servineo supports four distinct user roles:

### Requesters

<Card title="Requesters" icon="user">
  **Purpose**: Users who post service requests
  
  **Typical Percentage**: ~72% of total users
  
  **Capabilities**:
  - Create service requests
  - Browse fixer profiles
  - Schedule appointments
  - Rate and review fixers
</Card>

### Fixers

<Card title="Fixers" icon="wrench">
  **Purpose**: Service providers who fulfill requests
  
  **Typical Percentage**: ~27% of total users
  
  **Capabilities**:
  - Create professional profiles
  - Respond to service requests
  - Manage appointments
  - Build reputation through ratings
</Card>

### Visitors

<Card title="Visitors" icon="eye">

  **Capabilities**:
  - Browse public content
  - View fixer profiles
  - Access to limited features
</Card>

  **Capabilities**:
  - Full user management
  - Access to analytics
  - System configuration
  - Moderation tools
</Card>

## Refreshing Statistics

Admins can manually refresh user statistics:

```typescript /user-admin/components/adminDashboard.tsx
const handleRefreshStats = async () => {
  const token = localStorage.getItem('adminToken');
  if (!token) return;

  setStatsLoading(true);
  try {
    console.log('🔄 Updating statistics...');
    const statsResponse = await adminAPI.getUserStats(token);
    
    if (statsResponse.success && statsResponse.data) {
      setUserStats(statsResponse.data);
      setLastUpdated(new Date(statsResponse.data.timestamp).toLocaleTimeString());
      console.log('✅ Statistics updated:', statsResponse.data);
    }
  } catch (error) {
    console.error('❌ Error updating:', error);
  } finally {
    setStatsLoading(false);
  }
};
```

### Refresh Button UI

```tsx /user-admin/components/adminDashboard.tsx
<div className={styles.sectionHeader}>
```

## Error Handling

Robust error handling ensures data is always available:

<Steps>
  <Step title="Primary Data Source">
    Attempt to fetch real-time data from the API
  </Step>
  <Step title="Fallback on Error">
    Use cached/backup data if API fails
  </Step>
  <Step title="User Notification">
    Display informative note when using cached data
  </Step>
  <Step title="Automatic Recovery">
    Next refresh attempt will try real API again
  </Step>
</Steps>

```tsx /user-admin/components/adminDashboard.tsx
{userStats?.note && (
  <div className={styles.infoNote}>
    <div className={styles.infoIcon}>ℹ️</div>
    <div className={styles.infoContent}>
      <p>{userStats.note}</p>
      {userStats.source && <small>Source: {userStats.source}</small>}
    </div>
  </div>
)}
```

<Warning>
While fallback data keeps the dashboard functional, admins should investigate connectivity issues to ensure real-time accuracy.
</Warning>
