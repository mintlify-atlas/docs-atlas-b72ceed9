## Overview

Servineo Frontend implements a complete payment ecosystem supporting cash transactions, QR code payments, card processing, and a dedicated fixer wallet system. The payment system handles transactions, invoices, earnings, and bank account management.

### API Helper

The service uses a unified fetch wrapper that routes through the API proxy:

```typescript title="Unified API Fetch"
async function apiFetch<T = unknown>(
  path: string,
  init?: RequestInit & { json?: unknown }
): Promise<T> {
  const opts: RequestInit = {
    cache: "no-store",
    ...init,
    headers: { 
      "Content-Type": "application/json", 
      ...(init?.headers || {}) 
    },
  };
  
  if (init?.json !== undefined) {
    opts.body = JSON.stringify(init.json);
  }

  const res = await fetch(`/api${path}`, opts);

  if (!res.ok) {
    const raw = await res.text().catch(() => "");
    let msg = raw;
    const ct = res.headers.get("content-type") || "";
    
    if (ct.includes("application/json")) {
      try {
        const j = JSON.parse(raw);
        msg = j?.error || j?.message || raw || `HTTP ${res.status}`;
      } catch {}
    }
    
    throw new Error(`${msg || "Request failed"} (HTTP ${res.status})`);
  }

  const ct = res.headers.get("content-type") || "";
  return ct.includes("application/json") ? res.json() : ({} as T);
}
```

<Info>
All payment requests route through `/api` proxy to handle CORS and add authentication headers server-side.
</Info>

## Payment Operations

### Create Cash Payment

```typescript title="Create Cash Payment"
export type CreateCashPaymentDTO = {
  jobId: string;
  payerId?: string;
  requesterId?: string;
  fixerId?: string;
  subTotal: number;
  service_fee?: number;
  discount?: number;
  currency?: "BOB" | "USD";
  paymentMethods?: "cash" | "qr" | "card";
  commissionRate?: number;
};

export async function createCashPayment(input: CreateCashPaymentDTO) {
  const payload = {
    jobId: input.jobId,
    ...(input.requesterId ? { requesterId: input.requesterId } : {}),
    ...(input.fixerId ? { fixerId: input.fixerId } : {}),
    ...(input.payerId ? { payerId: input.payerId } : {}),
    subTotal: input.subTotal,
    service_fee: input.service_fee ?? 0,
    discount: input.discount ?? 0,
    currency: input.currency ?? "BOB",
    paymentMethods: input.paymentMethods ?? "cash",
    commissionRate: input.commissionRate ?? 0.1,  // 10% default
  };

  return apiFetch<{ message: string; data: BackendPaymentData }>(
    `/lab/payments`,
    { method: "POST", json: payload }
  );
}
```

### Response Normalization

The service normalizes varying backend response formats:

```typescript title="Payment Data Normalizer"
function normalizeSummary(d: BackendPaymentData | null | undefined): UISummary {
  if (!d) {
    return {
      id: "",
      status: "pending" as Status,
      code: null,
      expiresAt: null,
      amount: { total: 0, currency: "BOB" },
    };
  }

  const total = typeof d.total === "number"
    ? d.total
    : typeof d.amount?.total === "number"
    ? d.amount.total
    : 0;

  return {
    id: String(d.id ?? d._id ?? ""),
    status: (d.status ?? "pending") as Status,
    code: d.code ?? null,
    expiresAt: d.expiresAt ?? null,
    amount: {
      total,
      currency: d.amount?.currency ?? d.currency ?? "BOB",
    },
  };
}
```

### Fetching Fixer Data

```typescript title="Load Fixer Payment Data"
const fetchFixerData = async (fixerId: string) => {
  setLoading(true);
  setError(null);
  
  const BACKEND_URL = process.env.NEXT_PUBLIC_BACKEND_URL || 'http://localhost:8000';
  
  try {
    const response = await fetch(
      `${BACKEND_URL}/api/fixer/payment-center/${fixerId}`,
      {
        method: 'GET',
        headers: { 'Content-Type': 'application/json' },
      }
    );
    
    if (!response.ok) {
      const errData = await response.json();
      throw new Error(errData.error || `Error ${response.status}`);
    }
    
    const result = await response.json();
    
    if (result.success && result.data) {
      setFixerData(result.data);
    } else {
      throw new Error(result.error || 'Unknown error');
    }
  } catch (err: unknown) {
    console.error('Error fetching fixer data:', err);
    setError(err.message || 'Unknown error');
    
    // Fallback to test data
    setFixerData({
      saldoActual: 13.00,
      totalGanado: 15420.00,
      trabajosCompletados: 23,
      fixerId: fixerId, 
      isTestData: true 
    });
  } finally {
    setLoading(false);
  }
};
```

### Low Balance Alerts

The system automatically notifies fixers when their wallet balance is low:

```typescript title="Low Balance Toast Notifications"
useEffect(() => {
  const toastInfo = fixerData?.lowBalanceInfo?.toast;
  if (!toastInfo) return;

  const { level, message, shouldShow } = toastInfo;

  const baseOptions = {
    duration: 8000,
    position: "top-center" as const,
    transition: "bounceIn" as const,
    sound: false,
    progress: true,
  };

  if (shouldShow && level === "critical") {
    showToast.error(
      message || "Your balance is negative or critically low. Please recharge.",
      baseOptions
    );
    return;
  }

  if (shouldShow && level === "low") {
    showToast.warning(
      message || "Your balance is low. Consider recharging your wallet.",
      baseOptions
    );
    return;
  }

  if (level === "none") {
    showToast.success(
      message || "Your wallet is healthy.",
      baseOptions
    );
  }
}, [
  fixerData?.lowBalanceInfo?.toast?.shouldShow,
  fixerData?.lowBalanceInfo?.toast?.level,
]);
```

<Warning>
**Critical Balance Alert**: When balance drops below zero or a critical threshold, fixers receive an error-level toast notification.
</Warning>

## Payment Flows

### Cash Payment Flow

<Steps>
  <Step title="Job Completion">
    Fixer marks job as complete
  </Step>
  
  <Step title="Payment Initiation">
    System generates confirmation code
  </Step>
  
  <Step title="Cash Exchange">
    Requester pays fixer in person
  </Step>
  
  <Step title="Code Confirmation">
    Requester enters confirmation code in app
  </Step>
  
  <Step title="Payment Verification">
    Backend validates code and updates payment status
  </Step>
  
  <Step title="Wallet Credit">
    Fixer wallet is credited (minus commission)
  </Step>
</Steps>

### QR Payment Flow

<Steps>
  <Step title="Generate QR">
    System creates unique QR code with payment details
  </Step>
  
  <Step title="Display QR">
    QR shown to requester with amount and expiration
  </Step>
  
  <Step title="Scan & Pay">
    Requester scans with mobile wallet app
  </Step>
  
  <Step title="Payment Polling">
    Frontend polls backend for payment confirmation
  </Step>
  
  <Step title="Success Confirmation">
    Both parties notified of successful payment
  </Step>
</Steps>

## Commission System

Servineo takes a commission on each transaction:

```typescript

serviceFee = subTotal * commissionRate
```

## Best Practices

<Note>
1. **Never log sensitive data**: Card numbers, CVVs should never appear in logs
2. **Validate amounts server-side**: Always verify payment amounts on backend
3. **Use idempotency keys**: Prevent duplicate charges from retries
4. **Implement rate limiting**: Prevent payment spam and fraud
5. **Store minimal PCI data**: Use payment processor tokens instead of raw card data
</Note>
