## Overview

The Appointments API handles scheduling, tracking, and analytics for appointments between fixers and requesters. It includes admin endpoints for monitoring appointments across the platform.

## Endpoints

### Get Appointments by Fixer

Retrieve all appointments for a specific fixer.

```typescript
GET /api/appointments/fixer/{fixerId}
```

**Usage:**

```typescript
import { getAppointmentsByFixerId } from '@/services/appointments';

const appointments = await getAppointmentsByFixerId(fixerId);
```

**Path Parameters:**

<ParamField path="fixerId" type="string" required>
  The fixer's user ID
</ParamField>

**Response:**

<ResponseField name="_id" type="string">
  Appointment unique identifier
</ResponseField>

<ResponseField name="id_fixer" type="string">
  Fixer's user ID
</ResponseField>

<ResponseField name="id_requester" type="string">
  Requester's user ID
</ResponseField>

<ResponseField name="current_requester_name" type="string">
  Requester's display name
</ResponseField>

<ResponseField name="current_requester_phone" type="string">
  Requester's contact phone
</ResponseField>

<ResponseField name="appointment_type" type="string">
  Type of service appointment
</ResponseField>

<ResponseField name="appointment_description" type="string">
  Detailed description of the appointment
</ResponseField>

<ResponseField name="schedule_state" type="string">
  Current state: `booked`, `cancelled`, `completed`, `rescheduled`
</ResponseField>

<ResponseField name="selected_date" type="object">
  Date object with `$date` field (ISO 8601 string)
</ResponseField>

<ResponseField name="starting_time" type="object">
  Start time with `$date` field
</ResponseField>

<ResponseField name="finishing_time" type="object">
  End time with `$date` field
</ResponseField>

<ResponseField name="display_name_location" type="string">
  Human-readable location name
</ResponseField>

<ResponseField name="lat" type="string | null">
  Latitude coordinate
</ResponseField>

<ResponseField name="lon" type="string | null">
  Longitude coordinate
</ResponseField>

<ResponseField name="cancelled_fixer" type="boolean">
  Whether the fixer cancelled the appointment
</ResponseField>

<ResponseField name="reprogram_reason" type="string">
  Reason for rescheduling (if applicable)
</ResponseField>

<ResponseField name="link_id" type="string">
  Related link or reference ID
</ResponseField>

### Get Tracking Metrics

Retrieve appointment metrics with optional date filtering.

```typescript
GET /api/admin/metrics
```

**RTK Query Hook:**

```typescript
import { useGetTrackingMetricsQuery } from '@/app/redux/services/trackingAppointmentsApi';

// Without filters
const { data: metrics } = useGetTrackingMetricsQuery({});

// With date filters
const { data: filteredMetrics } = useGetTrackingMetricsQuery({
  startDate: '2024-01-01',
  endDate: '2024-01-31'
});
```

**Query Parameters:**

<ParamField query="startDate" type="string">
  Start date for filtering (ISO 8601 format: YYYY-MM-DD)
</ParamField>

<ParamField query="endDate" type="string">
  End date for filtering (ISO 8601 format: YYYY-MM-DD)
</ParamField>

**Response:**

<ResponseField name="total" type="number">
  Total number of appointments
</ResponseField>

<ResponseField name="active" type="number">
  Number of active (booked) appointments
</ResponseField>

<ResponseField name="cancelled" type="number">
  Number of cancelled appointments
</ResponseField>

<CodeGroup>
```typescript Response Type

  active: number;
  cancelled: number;
}
```


## Appointment States

Appointments can have the following states:

### Schedule States

<ParamField path="booked" type="state">
  Appointment is confirmed and scheduled
</ParamField>

<ParamField path="cancelled" type="state">
  Appointment has been cancelled by either party
</ParamField>

<ParamField path="completed" type="state">
  Service has been completed
</ParamField>

<ParamField path="rescheduled" type="state">
  Appointment has been rescheduled to a new time
</ParamField>

## Working with Dates

Appointment dates use MongoDB's extended JSON format:

```typescript
interface AppointmentTime {
  $date: string; // ISO 8601 format
}
```

**Parsing dates:**

```typescript
const appointment = await getAppointmentsByFixerId(fixerId);

appointment.forEach(apt => {
  const startDate = new Date(apt.starting_time.$date);
  const endDate = new Date(apt.finishing_time.$date);
  
  console.log(`Appointment from ${startDate.toLocaleString()} to ${endDate.toLocaleString()}`);
});
```

## Location Data

Appointments include location information in two formats:

1. **Coordinates**: Separate `lat` and `lon` fields (strings)
2. **Display Name**: Human-readable address in `display_name_location`

```typescript
const { lat, lon, display_name_location } = appointment;

if (lat && lon) {
  // Use coordinates for map display
  const position = {
    lat: parseFloat(lat),
    lng: parseFloat(lon)
  };
}

// Use display name for user-friendly address
console.log(`Location: ${display_name_location}`);
```

## Cache Tags

All appointment tracking endpoints use the `Statistics` cache tag:

- `getMapLocations`
- `getTrackingMetrics`
- `getFixerStats`

This tag is invalidated when appointment data changes, ensuring fresh statistics.

<Note>
The appointment fetching endpoint (`getAppointmentsByFixerId`) uses direct fetch calls rather than RTK Query, so it doesn't participate in the cache invalidation system. Always fetch fresh data when needed.
</Note>
