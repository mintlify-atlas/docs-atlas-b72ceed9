## Overview

Servineo Frontend implements a multi-layered notification system that provides user feedback through modal dialogs and toast notifications. The system supports various notification types, auto-dismiss functionality, and confirmation dialogs.

## Modal Notifications

The primary notification component is `NotificationModal` located at `src/Components/Modal-notifications.tsx`:

### Component Interface

```typescript title="NotificationModal Props"
interface NotificationModalProps {
  isOpen: boolean;
  onClose: () => void;
  type: 'success' | 'error' | 'info' | 'warning';
  title: string;
  message: ReactNode;
  icon?: LucideIcon;              // Custom icon override
  autoClose?: boolean;            // Default: true
  autoCloseDelay?: number;        // Default: 3000ms
  
  // Confirmation actions
  onConfirm?: () => void;
  onCancel?: () => void;
  confirmText?: string;           // Default: 'Confirmar'
  cancelText?: string;            // Default: 'Cancelar'
}
```

### Auto-Close Behavior

Notifications automatically close after a delay, except for warnings:

```typescript title="Auto-Close Logic"
const shouldAutoClose = isOpen && autoClose && type !== 'warning';

useEffect(() => {
  if (shouldAutoClose) {
    const timer = setTimeout(() => {
      onClose();
    }, autoCloseDelay);
    return () => clearTimeout(timer);
  }
}, [shouldAutoClose, autoCloseDelay, onClose]);
```

<Info>
**Warning notifications** do not auto-close by default to ensure users acknowledge important messages.
</Info>

### Progress Bar

Auto-closing notifications display an animated progress bar:

```tsx title="Progress Bar Component"
{shouldAutoClose && (
  <div className='mt-4 h-1 bg-gray-100 rounded-full overflow-hidden'>
    <div
      className={`h-full ${config.progressColor} animate-progress origin-left`}
      style={{ animationDuration: `${autoCloseDelay}ms` }}
    />
  </div>
)}
```

## Usage Examples

### Basic Success Notification

```typescript title="Success Message"
const [notification, setNotification] = useState<NotificationState>({
  isOpen: false,
  type: 'info',
  title: '',
  message: '',
});

// Show success
setNotification({
  isOpen: true,
  type: 'success',
  title: 'Payment Successful',
  message: 'Your payment of Bs. 150.00 has been processed.',
});

// In JSX
<NotificationModal
  isOpen={notification.isOpen}
  onClose={() => setNotification({ ...notification, isOpen: false })}
  type={notification.type}
  title={notification.title}
  message={notification.message}
/>
```

### Error Notification

```typescript title="Error Feedback"
setNotification({
  isOpen: true,
  type: 'error',
  title: 'Login Failed',
  message: 'Invalid credentials. Please check your email and password.',
});
```

### Warning with Confirmation

```typescript title="Confirmation Dialog"
setNotification({
  isOpen: true,
  type: 'warning',
  title: 'Delete Account',
  message: 'This action cannot be undone. Are you sure you want to delete your account?',
});

<NotificationModal
  isOpen={notification.isOpen}
  onClose={() => setNotification({ ...notification, isOpen: false })}
  type='warning'
  title='Delete Account'
  message='This action cannot be undone. Are you sure?'
  onConfirm={() => {
    // Handle deletion
    deleteAccount();
  }}
  onCancel={() => {
    // Just close
  }}
  confirmText='Delete'
  cancelText='Cancel'
  autoClose={false}
/>
```

## Toast Notifications

Servineo also uses `nextjs-toast-notify` for toast notifications:

```typescript title="Toast Notification Examples"
import { showToast } from "nextjs-toast-notify";

const baseOptions = {
  duration: 8000,
  position: "top-center" as const,
  transition: "bounceIn" as const,
  sound: false,
  progress: true,
};

// Success toast
showToast.success(
  "Operation completed successfully!",
  baseOptions
);

// Error toast
showToast.error(
  "An error occurred. Please try again.",
  baseOptions
);

// Warning toast
showToast.warning(
  "Your balance is low. Consider recharging.",
  baseOptions
);

// Info toast
showToast.info(
  "New features available in settings.",
  baseOptions
);
```

### Toast Positions

- `bottom-left`
- `bottom-center`
- `bottom-right`

## Notification State Management

### Notification State Interface

```typescript title="Notification State Type"
interface NotificationState {
  isOpen: boolean;
  type: 'success' | 'error' | 'info' | 'warning';
  title: string;
  message: string;
}

const initialState: NotificationState = {
  isOpen: false,
  type: 'info',
  title: '',
  message: '',
};
```

### Notification Helper Function

```typescript title="Reusable Notification Helper"
const notify = ({
  type,
  title,
  message,
}: Omit<NotificationState, 'isOpen'>) => {
  setNotification({
    isOpen: true,
    type,
    title,
    message,
  });
};

// Usage
notify({
  type: 'success',
  title: 'Saved',
  message: 'Your changes have been saved.',
});
```

## Best Practices

<Steps>
  <Step title="Be concise">
    Keep titles under 5 words and messages under 2 sentences
  </Step>
  
  <Step title="Use appropriate types">
    Match notification type to the action severity
  </Step>
  
  <Step title="Provide actionable feedback">
    Tell users what happened and what to do next
  </Step>
  
  <Step title="Don't overuse warnings">
    Reserve warnings for truly important confirmations
  </Step>
  
  <Step title="Test auto-close timing">
    Ensure users have enough time to read messages
  </Step>
</Steps>

## Common Patterns

### API Error Handling

```typescript title="Error Notification Pattern"
try {
  const result = await api.post('/endpoint', data);
  
  if (result.success) {
    notify({
      type: 'success',
      title: 'Success',
      message: result.message,
    });
  } else {
    notify({
      type: 'error',
      title: 'Operation Failed',
      message: result.error || 'An unknown error occurred.',
    });
  }
} catch (err) {
  notify({
    type: 'error',
    title: 'Connection Error',
    message: err.message || 'Could not connect to server.',
  });
}
```

### Form Validation

```typescript title="Validation Feedback"
const handleSubmit = (data) => {
  const errors = validate(data);
  
  if (errors.length > 0) {
    notify({
      type: 'warning',
      title: 'Validation Error',
      message: errors.join(', '),
    });
    return;
  }
  
  // Submit form
};
```

### Destructive Actions

```typescript title="Delete Confirmation"
const handleDelete = () => {
  setNotification({
    isOpen: true,
    type: 'warning',
    title: 'Confirm Deletion',
    message: 'This cannot be undone. Are you sure?',
  });
};

<NotificationModal
  {...notification}
  onConfirm={async () => {

    notify({
      type: 'success',
      title: 'Deleted',
      message: 'Item has been removed.',
    });
  }}
  confirmText='Delete'
  cancelText='Keep'
/>
```
