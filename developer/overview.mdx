## Welcome to Servineo Frontend

Servineo Frontend is a modern, full-stack web application built with Next.js 15, React 19, and Redux Toolkit. This platform connects service providers (fixers) with customers (requesters) through an intuitive job marketplace interface.

## Project Overview

The application is structured as a monorepo with the following key characteristics:

- **Framework**: Next.js 15.5.4 with App Router
- **React Version**: 19.1.0
- **Build Tool**: Turbopack (Next.js native)
- **Language**: TypeScript 5.9.3
- **State Management**: Redux Toolkit with RTK Query
- **Styling**: Tailwind CSS 4.1.17
- **Internationalization**: next-intl with English and Spanish support

<Info>
  This project uses the **Next.js 15 App Router** architecture, which is different from the legacy Pages Router. All routes are defined in the `src/app/[locale]/` directory.
</Info>

## Core Features

### Multi-Role System

The application supports two primary user roles:

1. **Requesters** - Customers who post job requests
2. **Fixers** - Service providers who fulfill job requests

### Key Functionality

- **Job Marketplace**: Browse, search, and filter service requests
- **Advanced Search**: Multi-criteria search with filters (src/app/[locale]/adv-search)
- **Appointment System**: Calendar-based scheduling with day/week/month views
- **Payment Integration**: Stripe integration for secure transactions
- **User Authentication**: Google OAuth and traditional login
- **Real-time Updates**: Redux-based state synchronization
- **Geolocation**: Leaflet maps for service area visualization
- **Multi-language**: English and Spanish with automatic locale detection

<Warning>

</Warning>

## API Architecture

The application uses a **proxy pattern** for API requests:

```typescript next.config.ts
async rewrites() {
  return [
    {
      source: '/api/:path*',
      destination: 'http://127.0.0.1:8000/api/:path*',
    },
  ];
}
```

All API calls to `/api/*` are proxied to the backend server running on port 8000. This is configured in:

- `next.config.ts` - Proxy rewrite rules
- `src/app/redux/services/baseApi.ts` - RTK Query base configuration
